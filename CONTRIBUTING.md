# Contributing to Under Construction

Thank you for your interest in contributing! This document provides guidelines for contributing.

## Getting Started

1. **Fork the Repository**
2. **Clone Your Fork**
   ```bash
   git clone https://github.com/YOUR-USERNAME/Under-connstruction.git
   cd Under-connstruction
   ```
3. **Create a Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Code Style

### HTML
- Use semantic HTML5 elements
- Meaningful class names
- Proper ARIA labels

### CSS
- Use CSS custom properties
- 2-space indentation
- Mobile-first approach

### JavaScript
- Use meaningful variable names
- Prefer const/let over var
- Add comments for complex logic
- Use arrow functions

## Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add countdown timer feature
fix: resolve animation timing issue
docs: update README
style: format CSS
refactor: improve particles system
perf: optimize animations
```

## Pull Request Process

1. **Make Your Changes**
   - Edit files
   - Test in multiple browsers
   - Keep commits focused

2. **Commit & Push**
   ```bash
   git add .
   git commit -m "feat: description"
   git push origin feature/your-feature
   ```

3. **Create Pull Request**
   - Go to original repository
   - Click "New Pull Request"
   - Fill in description
   - Submit!

## Testing Checklist

- [ ] Works in Chrome
- [ ] Works in Firefox
- [ ] Works in Safari
- [ ] Mobile responsive
- [ ] No console errors
- [ ] Performance acceptable
- [ ] Accessibility checked
